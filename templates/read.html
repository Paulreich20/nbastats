Current State of NBA Stats For you:

The Project uses a dataset that includes statistical averages for every NBA player from the 1984-1985 season until the 2016-2017 season. The main function of the website is to be able to search for any NBA player who played between 1984-1985 and then display their statistics. Furthermore, if the user wants to be able to compare two players directly, they can do so by using the Two Player Search, which will display both player’s statistics side by side.

Citations for Data set and loading data into database

Our data is a subset of a data se that we recieved from https://www.kaggle.com/drgilermo/nba-players-stats, a website for sharing open datasets, with licensing information found here: https://www.kaggle.com/terms. The data ranged from 1950 to 2018, but for the purpose of our project, we decided to only look at data from the 1984-1985 season. This subset was constructed using the R script located in our Repository.

Description of features:

The main two features of the website are searching for an individual player’s stats and searching for a pair of players whose stats to compare. Both search features have slightly different drop-down menus implemented using Javascript. When the user is typing in their query for an individual search, a drop-down menu will appear that lists the first three players sorted alphabetically that match their unfinished query. Clicking on a player will bring the user to that player’s stats page.

Once there, the user will be able to either navigate back to the home page of the website by clicking the logo or enter another search query using the navigation bar. Unfortunately, we were not able to implement the drop down-menus for the navigation bar without damaging the look and feel of the website.

When the user is typing in their query for a comparison search, there will again be a drop-down menu that lists the first three players sorted alphabetically that match their unfinished query. However, in this drop-down menu clicking on a player fills their name into the current search bar, which allows the user to use the drop-down menu for both players without annoyingly sending them to an individual player’s stats page.

Finally, if the user either has an empty input or an input that doesn’t match any NBA player when they submit their query, they will be taken to page informing them of their “bad” query and how to return to the homepage.

One More Feature

One feature we wish we could've implemented if we had more time was to have a team search. The idea would be to search for a team and then depending on a user selected stat display the top 5 or 10 players who had the highest averages of that selected stat. For example, If I searched the Cleveland Cavaliers and selected points, I would likely see several of Lebron's high scoring seasons as well as some of Kyrie Irving's.
